GitHub
service in module gitphaser

Description

Provides access to the GitHub API

Methods

  • authenticate()

    Logs user into GitHub via inAppBroswer. Sets authToken

    Returns
    Promise

    Resolves on success, rejects if $cordovaOauth fails.

  • canFollow(username)

    Searches the users following list to see if target is already followed.

    Parameters
    ParamTypeDetails
    username string

    GitHub login name of the target to follow.

    Returns
    boolean

    Returns true if user can follow person, false otherwise.

  • follow(username)

    Attempts to follow the user specified by param username. Increments follower/following metrics and adds a mock follower to the followers array to keep cache current w/ GitHub remote.

    Parameters
    ParamTypeDetails
    username string

    GitHub login name of the target to follow.

    Returns
    promise

    Resolves on success, rejects on error

  • getAccount(username, auth_api)

    Gets GitHub profile, public repos and public events from GitHub for an arbitrary user

    Parameters
    ParamTypeDetails
    username string

    User's github login name

    auth_api object

    GitHub.js api initialized with users oAuth credentials

    Returns
    Promise

    Resolves account object on success.

  • getAuthToken()

    Convenience method to get authToken when it needs to be saved in user account.

    Returns
    String

    authToken Github token acquired by user during oAuth login

  • getContribGraph(username)

    Retrieves svg string representing a contributions graph from cache or by having the Meteor server fetch it.

    Parameters
    ParamTypeDetails
    username string

    Any github login name

    Returns
    Promise

    Resolves svg string.

  • getMe()

    Collects user's profile info, repos, events and followers from Github.

    Returns
    Promise

    Resolves on success.

  • initialize()

    Invoked in the routing resolve at tab/nearby - if user autologs into Meteor, we still need to fetch a fresh GitHub profile for them. Only initializes if self.me doesn't exist yet.

    Returns
    Promise

    Resolves when github responds with profile, rejects with 'AUTH_REQUIRED'

  • setAuthToken(Github)

    Convenience method to set authToken when app is already authenticated from previous use.

    Parameters
    ParamTypeDetails
    GithubString

    token acquired by user during oAuth login

Properties

  • api

    Object: GitHub.js api

  • cache

    Array: Cached account object and contrib graphs fetched from Github

  • cache_time

    String: Duration to cache for. Options are: second, minute, hour, week

  • me

    Object: The user's profile

  • repos

    Array: The users public repos
